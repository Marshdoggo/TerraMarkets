# 🌎 TerraMarkets: Decentralized Prediction Markets for Earth, Climate, and Biodiversity

**TerraMarkets** is a web-based prediction market app designed for Earth-conscious forecasting — including topics like climate policy, biodiversity events, natural disasters, and sustainable technology adoption. Built using a modern Python backend stack, it simulates core mechanics of platforms like Kalshi and Polymarket while staying flexible enough to pivot between centralized and decentralized approaches.

---

## ✅ Project Goals
- Enable users to bet on binary outcome markets (Yes/No)
- Track user balances and market outcomes
- Ensure security with password hashing and JWT-based authentication
- Build a backend that's modular, testable, and frontend-ready

---

## 🧱 Architecture Overview

### ⚙️ Backend: FastAPI + SQLite + SQLAlchemy
- **FastAPI**: for high-performance Python APIs
- **SQLite**: local dev-friendly database
- **SQLAlchemy ORM**: for clean, Pythonic database models
- **JWT Auth**: using `python-jose` for token generation and validation
- **Password Security**: using `passlib[bcrypt]` to hash and verify passwords

### 📁 Project Structure
```
TerraMarkets/
├── backend/
│   ├── main.py                # FastAPI app + routes
│   ├── database.py            # DB engine, Base, UserDB model, get_db()
│   ├── auth.py                # JWT auth, token creation, user validation
│   ├── security.py            # Password hashing + verification
│   ├── models.py              # SQLAlchemy models (Markets, Bets)
│   ├── schemas.py             # Pydantic models for FastAPI validation
├── contracts/                 # Optional: Smart contracts (if blockchain path is revisited)
├── frontend/                  # Placeholder for future React UI
├── terra.db                   # Local SQLite DB (autogenerated)
├── requirements.txt           # Project dependencies
```

### 🔐 Auth Flow
- `POST /register`: create new user (hashed password saved to DB)
- `POST /token`: login to receive JWT access token
- `GET /me`: authenticated route (requires Bearer token)

---

## 🛠️ API Features
- `POST /register`: Create a new user
- `POST /token`: Log in and get JWT
- `GET /me`: View your user profile (auth-only)
- `POST /markets`: Create a binary market ("Will X happen?")
- `GET /markets`: List all open/closed markets
- `POST /bet`: Place a Yes/No bet on a market
- `POST /resolve/{market_id}`: Resolve a market to Yes or No
